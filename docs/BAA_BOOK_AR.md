## 1. مقدمة

لغة **باء (Baa)** هي لغة برمجة مُترجمة بأسلوب عربي، تستهدف حالياً **ويندوز x86-64** و**لينكس x86-64** عبر توليد ملف تجميع (`.s`) ثم استخدام **gcc** المضيف للتجميع والربط.

هذا الكتاب يهدف إلى أن يكون:

- دليلًا تعليميًا يبدأ من الأساسيات إلى الميزات المتقدمة ضمن نطاق **v0.3.10.6**
- مرجعًا سريعًا للكلمات المفتاحية والمعاملات وصياغة الأوامر
- موردًا عمليًا يحتوي على أمثلة صحيحة وتمارين للتثبيت

**الميزات الرئيسية في v0.3.10.6:**
- مؤشرات الدوال (`دالة(...) -> ...`)
- المؤشرات العامة مع العمليات الحسابية
- التحويل الصريح بين الأنواع (`كـ<نوع>(...)`)
- الأنواع الصحيحة ذات الأحجام المحددة
- الهياكل والاتحادات والتعدادات
- المصفوفات أحادية ومتعددة الأبعاد
- المعالج القبلي الكامل

## 2. التهيئة وبناء المترجم

### المتطلبات (Windows)

- CMake 3.10+
- MinGW-w64 (GCC) ويُفضّل أن يكون `gcc` على `PATH`
- PowerShell
- Git

### المتطلبات (Linux)

- CMake 3.10+
- GCC أو Clang
- Git

### البناء من المصدر

**Windows:**

```powershell
git clone https://github.com/OmarAglan/Baa.git
cd Baa
cmake -B build -G "MinGW Makefiles"
cmake --build build
```

**Linux:**

```bash
git clone https://github.com/OmarAglan/Baa.git
cd Baa
cmake -B build-linux -DCMAKE_BUILD_TYPE=Release
cmake --build build-linux -j
```

بعد البناء ستجد `baa.exe` (Windows) أو `baa` (Linux) داخل مجلد البناء.

## 3. البرنامج الأول

احفظ الملف بترميز **UTF-8**.

```baa
// برنامجي الأول في باء
صحيح الرئيسية() {
    اطبع "مرحباً بالعالم!".
    إرجع ٠.
}
```

الترجمة والتنفيذ (من داخل مجلد `build` مثلًا):

```powershell
# Windows
.\baa.exe ..\hello.baa -o hello.exe
.\hello.exe

# Linux
./build-linux/baa ../hello.baa -o hello
./hello
```

## 4. الكتابة الصحيحة: الترميز وعلامات الترقيم

### 4.1. الترميز (Encoding)

- الملفات يجب أن تكون **UTF-8** حتى تظهر الكلمات العربية بشكل صحيح داخل المترجم.
- يدعم المترجم ملفات ذات BOM أو بدونه.

### 4.2. علامة نهاية الجملة

- كل جملة تنتهي بنقطة `.` (بديلًا عن `;` في C).

### 4.3. فاصلة for العربية

- حلقة `لكل` تستخدم الفاصلة المنقوطة العربية **`؛`** للفصل داخل الأقواس.

مثال صحيح:

```baa
لكل (صحيح س = ٠؛ س < ٣؛ س++) {
    اطبع س.
}
```

## 5. الأنواع والمتغيرات والثوابت

### 5.1. الأنواع الأساسية (v0.3.10.6)

| النوع | الوصف | الحجم |
|-------|-------|-------|
| `صحيح` | عدد صحيح موقع | 64 بت (بديل عن `ص٦٤`) |
| `ص٨` | عدد صحيح موقع 8 بت | 8 بت |
| `ص١٦` | عدد صحيح موقع 16 بت | 16 بت |
| `ص٣٢` | عدد صحيح موقع 32 بت | 32 بت |
| `ص٦٤` | عدد صحيح موقع 64 بت | 64 بت |
| `ط٨` | عدد صحيح غير موقع 8 بت | 8 بت |
| `ط١٦` | عدد صحيح غير موقع 16 بت | 16 بت |
| `ط٣٢` | عدد صحيح غير موقع 32 بت | 32 بت |
| `ط٦٤` | عدد صحيح غير موقع 64 بت | 64 بت |
| `عشري` | عدد فاصلة عائمة | 64 بت (f64) |
| `حرف` | محرف UTF-8 | 64 بت |
| `نص` | سلسلة محارف | مؤشر (`حرف[]`) |
| `منطقي` | قيمة منطقية | 8 بت |
| `عدم` | بدون قيمة (للدوال) | - |

القيم المنطقية: `صواب` (true) و `خطأ` (false).
المؤشر الفارغ: `عدم` (null).

### 5.2. تعريف متغير (محلي)

تعريف المتغير المحلي المتضمن للتهيئة:

```baa
صحيح الرئيسية() {
    صحيح س = ١٠.
    نص رسالة = "باء".
    منطقي جاهز = صواب.
    عشري قيمة = ٣.١٤.
    إرجع ٠.
}
```

### 5.3. المتغيرات العامة (Global)

المتغير العام يمكن أن يُعرّف بقيمة ابتدائية أو بدونها:

```baa
صحيح س.           // قيمة افتراضية (صفر)
صحيح ص = ١٠.
نص اسم = "علي".
```

### 5.4. الثوابت: `ثابت`

الثابت لا يمكن إعادة إسناده بعد التهيئة:

```baa
صحيح الرئيسية() {
    ثابت صحيح حد = ١٠٠.
    // حد = ٢٠٠.  // خطأ دلالي
    اطبع حد.
    إرجع ٠.
}
```

### 5.5. أحجام الأعداد الصحيحة (v0.3.6)

لبرمجة النظم الدقيقة، نحتاج لتحديد حجم الذاكرة بدقة. توفر باء أنواعًا بأحجام محددة:

- **موقعة:** `ص٨`, `ص١٦`, `ص٣٢`, `ص٦٤` (مثل `int32_t` وغيرها)
- **غير موقعة:** `ط٨`, `ط١٦`, `ط٣٢`, `ط٦٤` (مثل `uint32_t`)

مثال:
```baa
ص٨ بايت_صغير = ١٠٠.
ط٦٤ عداد_ضخم = ٥٠٠٠٠٠٠٠٠٠.
```

### 5.6. أسماء الأنواع البديلة: `نوع` (v0.3.6)

يمكنك إنشاء اسم جديد لنوع موجود لتسهيل القراءة:

```baa
نوع معرف_مستخدم = ط٦٤.
نوع كود_خطأ = ص٣٢.

معرف_مستخدم م = ١٢٣٤٥.
```

**ملاحظات:**
- تعريفات `نوع` تكون عامة (top-level) فقط.
- يجب إعلان الاسم البديل قبل استخدامه.

### 5.7. التحويل الصريح: `كـ` (v0.3.10.5)

للتحويل الصريح بين الأنواع (Casting) نستخدم الصيغة `كـ<النوع>(القيمة)`:

```baa
صحيح عدد = ٦٥.
حرف رمز = كـ<حرف>(عدد).

عشري ف = ٣.٩٩.
صحيح س = كـ<صحيح>(ف).  // س = ٣
```

**التحويلات المدعومة:**
- التحويلات العددية: بين `صحيح`، `عشري`، `حرف`
- تحويلات المؤشرات: مؤشر ↔ مؤشر، مؤشر ↔ `صحيح`

### 5.8. المؤشرات: `*` و `&` (v0.3.10)

المؤشرات تسمح بالتعامل المباشر مع عناوين الذاكرة:

- **أخذ العنوان (`&`)**: الحصول على عنوان متغير.
- **فك الإشارة (`*`)**: الوصول للقيمة الموجودة في عنوان.
- **عدم**: تستخدم لتمثيل المؤشر الفارغ (Null Pointer).

```baa
صحيح س = ٤٢.
صحيح* ص = &س.  // ص يشير إلى عنوان س
*ص = ١٠٠.       // تغيير قيمة س عبر المؤشر
ص = عدم.        // ص الآن مؤشر فارغ
```

**العمليات على المؤشرات:**
- الجمع/الطرح بالأعداد: `مؤشر + عدد`، `مؤشر - عدد`
- طرح مؤشر من مؤشر: `مؤشر - مؤشر` (يعيد عدد العناصر)
- المقارنة: `==`، `!=` (مع السماح بـ `عدم`)

### 5.9. مؤشرات الدوال: `دالة` (v0.3.10.6)

يمكن تمرير الدوال كقيم باستخدام توقيع الدالة كنوع:

**الصيغة:** `دالة(أنواع_المعاملات) -> نوع_الرجوع`

```baa
نوع عملية = دالة(صحيح، صحيح) -> صحيح.

صحيح جمع(صحيح أ، صحيح ب) { إرجع أ + ب. }

صحيح الرئيسية() {
    عملية ع = جمع.
    اطبع ع(١، ٢).  // يطبع ٣
    
    // مقارنة مع عدم
    إذا (ع != عدم) {
        اطبع "المؤشر صالح".
    }
    
    إرجع ٠.
}
```

**الاستخدام كمعاملات:**

```baa
نوع معالج = دالة(صحيح) -> عدم.

عدم طبق_معالج(صحيح قيمة، معالج م) {
    م(قيمة).
}

عدم اطبع_ضعف(صحيح س) {
    اطبع س * ٢.
}

صحيح الرئيسية() {
    طبق_معالج(٥، اطبع_ضعف).  // يطبع ١٠
    إرجع ٠.
}
```

### 5.10. الاتحادات: `اتحاد` (v0.3.4.5)

الاتحاد (`Union`) يسمح بتخزين أنواع مختلفة في نفس موقع الذاكرة.

```baa
اتحاد بيانات {
    صحيح رقم.
    نص نص_قيمة.
}

اتحاد بيانات د.
د:رقم = ٥٠.

// الكتابة إلى عضو آخر تكتب فوق نفس الذاكرة
د:نص_قيمة = "مرحبا".
اطبع د:نص_قيمة.
```

### 5.11. المتغيرات الساكنة: `ساكن` (v0.3.7.5)

المتغير الساكن داخل الدالة يحتفظ بقيمته بين الاستدعاءات:

```baa
صحيح عداد() {
    ساكن صحيح ع = ٠.
    ع = ع + ١.
    إرجع ع.
}

صحيح الرئيسية() {
    اطبع عداد().  // ١
    اطبع عداد().  // ٢
    اطبع عداد().  // ٣
    إرجع ٠.
}
```

**ملاحظات:**
- `ساكن` مدعوم للمتغيرات والمصفوفات العامة والمحلية.
- التهيئة غير الثابتة وقت الترجمة مرفوضة للتخزين الساكن.
- إن غابت التهيئة، القيمة الافتراضية هي الصفر.

## 6. التعابير والمعاملات

### 6.1. معاملات حسابية

- `+ - * / %`
- `++ --` (بعدي postfix)

```baa
صحيح الرئيسية() {
    صحيح س = ١٠.
    س++.
    اطبع س.
    إرجع ٠.
}
```

### 6.2. المقارنات والمنطق

- المقارنة: `== != < > <= >=`
- المنطق: `&& || !` مع **تقييم قصير** (short-circuit)

```baa
صحيح الرئيسية() {
    صحيح س = ٥.
    إذا (س > ٠ && س < ١٠) {
        اطبع "ضمن المجال".
    }
    إرجع ٠.
}
```

### 6.3. العمليات منخفضة المستوى (v0.3.6)

- معاملات البت: `& | ^ ~`
- الإزاحة: `<< >>`

```baa
صحيح الرئيسية() {
    صحيح أ = ٥ & ٣.   // ١
    صحيح ب = ٥ | ٣.   // ٧
    صحيح ج = ٥ ^ ٣.   // ٦
    صحيح د = ~٥.      // bitwise NOT
    صحيح هـ = ١ << ٤. // ١٦
    إرجع ٠.
}
```

### 6.4. معامل الحجم: `حجم`

يُستخدم لمعرفة حجم النوع أو التعبير بالبايت:

```baa
صحيح حجم_الصحيح = حجم(صحيح).  // ٨ على x86-64
صحيح حجم_الحرف = حجم(حرف).     // ٨
```

## 7. التدفق المنطقي: إذا/وإلا والحلقات

### 7.1. إذا/وإلا

```baa
صحيح الرئيسية() {
    صحيح عمر = ١٨.
    إذا (عمر >= ١٨) {
        اطبع "بالغ".
    } وإلا إذا (عمر >= ١٣) {
        اطبع "مراهق".
    } وإلا {
        اطبع "قاصر".
    }
    إرجع ٠.
}
```

### 7.2. طالما (while)

```baa
صحيح الرئيسية() {
    صحيح س = ٣.
    طالما (س > ٠) {
        اطبع س.
        س = س - ١.
    }
    إرجع ٠.
}
```

### 7.3. لكل (for)

```baa
صحيح الرئيسية() {
    لكل (صحيح س = ٠؛ س < ٣؛ س++) {
        اطبع س.
    }
    إرجع ٠.
}
```

### 7.4. توقف/استمر

```baa
صحيح الرئيسية() {
    لكل (صحيح س = ٠؛ س < ١٠؛ س++) {
        إذا (س == ٥) {
            استمر.  // تخطي هذه التكرارة
        }
        إذا (س == ٨) {
            توقف.   // الخروج من الحلقة
        }
        اطبع س.
    }
    // الناتج: ٠ ١ ٢ ٣ ٤ ٦ ٧
    إرجع ٠.
}
```

### 7.5. اختر (switch)

```baa
صحيح الرئيسية() {
    صحيح س = ٢.
    اختر (س) {
        حالة ١:
            اطبع "واحد".
            توقف.
        حالة ٢:
            اطبع "اثنان".
            توقف.
        افتراضي:
            اطبع "غير ذلك".
            توقف.
    }
    إرجع ٠.
}
```

## 9. المصفوفات

### 9.1. تعريف المصفوفة

المصفوفات مدعومة للأنواع المختلفة وبأحجام ثابتة:

```baa
صحيح قائمة[٥].
قائمة[٠] = ١٠.
قائمة[١] = ٢٠.
```

### 9.2. المصفوفات متعددة الأبعاد

```baa
صحيح جدول[٢][٣].
جدول[١][٢] = ٧.
صحيح قيمة = جدول[١][٢].
```

### 9.3. تهيئة المصفوفة

يمكن تهيئتها مباشرة باستخدام الأقواس المعقوفة `{...}`:

```baa
صحيح قائمة[٥] = {٠، ١٠، ٢٠، ٣٠، ٤٠}.

// تهيئة متداخلة (مصفوفة ثنائية)
صحيح م[٢][٣] = {{١، ٢، ٣}، {٤}}.

// مصفوفة نصوص
نص أسماء[٢] = {"علي"، "منى"}.
حرف ح = أسماء[٠][١].
```

### 9.4. حساب طول المصفوفة

يُحسب طول المصفوفة باستخدام الصيغة:

```baa
صحيح قائمة[١٠].
صحيح الطول = حجم(قائمة) / حجم(صحيح).  // ١٠
```

## 10. الهياكل والتعدادات

### 10.1. التعدادات: `تعداد`

التعداد يعرّف مجموعة أسماء ثابتة تُخزَّن كأعداد صحيحة:

```baa
تعداد لون {
    أحمر،
    أزرق،
    أسود،
}

صحيح الرئيسية() {
    تعداد لون ل = لون:أحمر.
    إذا (ل == لون:أحمر) {
        اطبع "اللون أحمر".
    }
    إرجع ٠.
}
```

**ملاحظات:**
- قيم العناصر تُعيَّن تلقائياً: 0, 1, 2...
- لا يمكن مقارنة/إسناد قيم تعداد من أنواع تعداد مختلفة.

### 10.2. الهياكل: `هيكل`

الهيكل يعرّف نوعاً مركّباً من حقول (Fields):

```baa
هيكل نقطة {
    صحيح س.
    صحيح ص.
}

صحيح الرئيسية() {
    هيكل نقطة ن.
    ن:س = ١.
    ن:ص = ٢.
    اطبع ن:س.
    إرجع ٠.
}
```

**التعشيش:**

```baa
هيكل محرك { صحيح قوة. }

هيكل سيارة {
    هيكل محرك م.
}

صحيح الرئيسية() {
    هيكل سيارة س.
    س:م:قوة = ١٥٠.
    اطبع س:م:قوة.
    إرجع ٠.
}
```

**قيود حالية:**
- لا يمكن استخدام الهيكل كقيمة من الدرجة الأولى (لا يوجد إسناد/تمرير/إرجاع بالنسخ).
- تهيئة الهيكل عند التعريف غير مدعومة حالياً.

## 11. الإدخال/الإخراج: اطبع واقرأ

### 11.1. اطبع

- يطبع قيمة ثم سطرًا جديدًا.
- يدعم: الأعداد الصحيحة (جميع الأحجام)، `عشري`، `نص`، `حرف`، `منطقي`.

```baa
اطبع "مرحباً".
اطبع ١٠٠.
اطبع ٣.١٤.
```

### 11.2. اقرأ

- يقرأ عددًا صحيحًا من الإدخال القياسي إلى متغير.
- يدعم جميع أنواع الأعداد الصحيحة.

```baa
صحيح الرئيسية() {
    صحيح س = ٠.
    اقرأ س.
    اطبع س.
    إرجع ٠.
}
```

## 14. المعالج القبلي (Preprocessor)

اللغة تدعم التوجيهات التالية:

| التوجيه | الوصف |
|---------|-------|
| `#تضمين "file"` | تضمين ملف |
| `#تعريف الاسم قيمة` | تعريف ماكرو |
| `#إذا_عرف الاسم` | ترجمة شرطية (إذا معرف) |
| `#وإلا` | الجزء البديل للترجمة الشرطية |
| `#نهاية` | نهاية الترجمة الشرطية |
| `#الغاء_تعريف الاسم` | إلغاء تعريف ماكرو |
| `#خطأ رسالة` | إنشاء خطأ ترجمة |

مثال:

```baa
#تعريف حد ٣

صحيح الرئيسية() {
#إذا_عرف حد
    اطبع حد.
#وإلا
    اطبع ٠.
#نهاية
    إرجع ٠.
}
```

مثال متقدم:

```baa
#تعريف النسخة ١

#إذا_عرف النسخة
    #إذا (النسخة < ٢)
        #خطأ "النسخة قديمة جداً"
    #نهاية
#وإلا
    #تعريف النسخة ١
#نهاية
```

## 15. التشخيص: الأخطاء والتحذيرات

### 15.1. الأخطاء

الأخطاء النحوية تُعرض بصيغة:

`[Error] file:line:col: <message>` مع سطر من المصدر وسهم `^`.

### 15.2. التحذيرات

التحذيرات تُفعل بـ `-Wall` أو أعلام محددة:

| العلم | التحذير |
|-------|---------|
| `-Wunused-variable` | متغير معرف لكن غير مستخدم |
| `-Wdead-code` | كود بعد إرجع أو توقف |
| `-Wimplicit-return` | دالة بدون إرجاع صريح |
| `-Wshadow-variable` | متغير محلي يحجب متغير عام |
| `-Wimplicit-narrowing` | تحويل ضمني قد يسبب فقدان بيانات |
| `-Wsigned-unsigned-compare` | مقارنة بين موقّع وغير موقّع |

يمكن تحويل التحذيرات إلى أخطاء بـ `-Werror`.

## 16. الأداء وأفضل الممارسات

### 16.1. نصائح أداء

- تجنب التعابير المكثفة داخل حلقات كبيرة إذا أمكن.
- انتبه إلى الاستدعاء التكراري (Recursion) لأن عمق الاستدعاء قد يؤدي إلى تجاوز المكدس.
- استخدم `ثابت` للقيم الثابتة بدلاً من المتغيرات.
- استخدم `ساكن` للمتغيرات التي تحتاج للاحتفاظ بالقيمة بين الاستدعاءات.

### 16.2. أخطاء شائعة

| الخطأ | الحل |
|-------|------|
| نسيان النقطة `.` | كل جملة تنتهي بنقطة |
| استخدام `;` بدل `؛` | حلقة `لكل` تستخدم الفاصلة العربية |
| تعريف متغير بدون تهيئة | المتغيرات المحلية يجب تهيئتها |
| خلط أنواع التعدادات | لا يمكن مقارنة تعدادين مختلفين |
